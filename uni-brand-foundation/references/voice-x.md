# Voice — X / Twitter

Extends `voice-core.md`. Use for protocol announcements, devrel posts, and threads.

## 1. Post types
- **Single drop** — one tight claim, one number, one link. <240 chars.
- **Thread** — 3–9 tweets. Set up, payoff, proof, call to action.
- **Reply chain** — joining a relevant convo without hijacking it.

## 2. Opener patterns we use
- **The unexpected number.** Sample: *"v4 hooks have shipped 1,400+ unique deployments since mainnet."*
- **The named scenario.** Sample: *"Your treasury just hit $40M in stables. The board wants yield. The CFO wants safety."*
- **The technical reframe.** Sample: *"Most people think a hook is a smart contract. It's a callback. That distinction changes the architecture."*

## 3. Opener patterns we avoid
- "GM" / "WAGMI" openers — off-brand for the protocol layer
- "Massive update 🚨" / siren emojis
- "Excited to announce..."
- Threads that take 4 tweets to get to the point

## 4. Length and structure
- Single posts: <240 chars (room to be quoted).
- Threads: tweet 1 must stand alone if nobody clicks the thread.
- One link per thread, in the final tweet.

## 5. Visual conventions
- Numbers as digits, always.
- One unicorn emoji 🦄 maximum, only if structurally meaningful.
- No hashtag spam. One project hashtag if relevant; never trend-chasing.
